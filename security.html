Security

Last updated: May 12, 2026

Core principle: Matchic's security model is built on a simple premise — if data never leaves your device, it cannot be intercepted, breached, or misused server-side.

Architecture

Matchic follows a zero-server architecture. The app contains no server-side component, no cloud database, and no user accounts. All functionality — including AI inference — runs locally on your device using on-device machine learning models.

Data at Rest

Your wardrobe data is stored exclusively on your device:

  • ObjectBox database: Structured data (items, tags, categories, outfits) is stored in a local NoSQL database within the app's private sandboxed directory
  • Photo storage: Images are stored in the app's private file directory, inaccessible to other applications
  • Preferences: App settings are stored via SharedPreferences/NSUserDefaults in the app's sandbox

On modern iOS (iOS 8.0+) and Android (Android 7.0+) devices, data at rest is protected by:

  • File-based encryption (FBE): The operating system encrypts each file with a per-file key
  • App sandboxing: Each app runs in its own sandbox with restricted access to system resources
  • Device lock screen: Data is only accessible when the device is unlocked

Data in Transit

Matchic does not transmit any wardrobe data over the network. The only network requests the app makes are:

  • AI model download: When you explicitly choose to enable AI features, the Gemma 3 model (~275MB) is downloaded via HTTPS from Google's official repository. This is a standard file download — no personal data is included or transmitted.
  • Font rendering: If using Google Fonts, font files may be cached locally on first use (applies to our website, not the app itself)

No data is ever sent to our servers because we have no servers. There is no telemetry, no analytics, no crash reporting that transmits personal data.

AI Model Security

The on-device AI model (Gemma 3) is downloaded from Google's official repository via HTTPS. Once downloaded:

  • The model file is stored in the app's private directory
  • All inference runs offline — no data is sent to Google or any third party
  • The model itself is a static file that performs no network communication
  • You can delete the model at any time via Settings or by clearing app data

Backup Security

The Backup & Restore feature exports your wardrobe as a ZIP file. Important security notes:

  • Backup files are saved to your device's shared storage (Downloads folder) and are accessible to other apps with storage permissions
  • We recommend not storing backup files on cloud storage services unless you trust their security
  • Backup files contain your actual clothing photos and metadata — treat them as sensitive personal data
  • We are working on adding encryption to backup files in a future update
  • Always delete backup ZIP files after restoring, especially if created on a shared device

Dependencies & Supply Chain

All third-party libraries are sourced from trusted package registries (pub.dev) with verified publisher identities. Our dependencies are:

  • Pinned to specific versions in pubspec.lock to prevent supply chain attacks
  • Regularly reviewed for known vulnerabilities via flutter pub outdated
  • Sourced exclusively from the official Dart/Flutter package ecosystem

Vulnerability Disclosure

If you discover a security vulnerability in Matchic, please report it responsibly:

  • Email: support@matchic.app with subject "Security Vulnerability"
  • GitHub: Report via Security Advisories
  • PGP key: Available upon request

We aim to acknowledge receipt within 48 hours and provide a fix timeline within 7 days. We practice coordinated disclosure and will credit researchers who report vulnerabilities.

Bug Bounty

At this time, Matchic does not operate a formal bug bounty program. However, significant security improvements suggested by researchers may be acknowledged with credit and attribution in our release notes.

Security Checklist for Users

  • ✅ Use a strong device passcode or biometric lock
  • ✅ Keep your device OS updated to the latest version
  • ✅ Regularly export and encrypt backups
  • ✅ Delete backup ZIP files after restoring
  • ✅ Only download Matchic from official app stores (Google Play, Apple App Store)
  • ✅ Do not install Matchic from third-party APK sources
